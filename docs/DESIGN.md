# デザイン解説 — Retro Computer Website (edh.dev)

このドキュメントは、本サイトの**ビジュアル・デザイン言語**を言語化したものです。
（three.js / UNIXシェル再現などのロジック部分は対象外）

このサイトは「3Dレトロコンピュータ（Commodore PET 8296風）のWebGLヒーローセクション」
+「ジン/フリーペーパー風のフラットなドキュメントセクション」という2層構造を持っています。
以下はその"見た目"の設計原則です。

---

## 1. コンセプト：レトロPC / ジン (Zine) ハイブリッド

- 画面上部はWebGLで描画された**80年代パーソナルコンピュータの3Dモデル**（実機: Commodore PET 8296）が主役。
- スクロールすると3D演出からフラットな本文セクションへ移行し、紙のZineやリスクグラフ印刷物のような
  **手作り感・コピー機感のあるレイアウト**になる。
- 「コンピュータの中の世界」と「印刷物としてのポートフォリオ」を1ページに同居させているのが核。

---

## 2. カラーパレット

固定の2色構成（モノクロ印刷のZineのような配色）。

| 用途 | 色 | 備考 |
|---|---|---|
| 背景（ベース） | `#f6d4b1`（クリーム/淡いベージュ・オレンジ） | ローディング画面の文字色にも使用、`theme-color`にも指定 |
| 前景・テキスト・枠線 | `#525252`（ダークグレー） | CSS変数 `--color` として全体管理。純黒ではなく中間グレーで柔らかさを出す |
| 見出しの背景反転 | `--color` を背景、ベージュを文字色にする「ネガ表示」 | h1, h2, ナビボタンなどに多用 |

- **純白・純黒を使わない**のが特徴。常にベージュ×グレーの2トーンで統一し、
  古い印刷物・CRTモニタのような落ち着いた色温度を演出。
- 影（shadow）にもこの2色しか使わない（後述）。

---

## 3. タイポグラフィ

2種類のカスタムウェブフォントを役割分担。

1. **`public-pixel`**（ピクセルフォント、monospace）
   - 見出し（h1, h2, h3）、ナビゲーションメニュー、ボタンラベルなど「コンピュータUI」を感じさせる箇所に使用。
   - 角ばった8bit/レトロターミナル感を出す。
2. **`chill`**（サンセリフ）
   - 本文（p, リストなど）の可読性重視フォント。
   - ピクセルフォントと対比させることで「ターミナルの見出し」と「読みやすい本文」のメリハリを作る。

見出し（h1/h2）は**背景色を塗った"ラベルタグ"風**（パディング+背景色反転+インラインブロック）で表示し、
紙に貼った付箋やスタンプのように見せている。

---

## 4. シャドウ／枠線の作法（「コピー用紙の浮き出し」スタイル）

このサイト全体を貫く最も特徴的なディテール。

```
box-shadow: 1px 1px 0px <ベージュ>, 6px 6px 0px rgba(グレー, 0.25);
```

- **二重シャドウ**：1px の濃い縁取り＋6px のオフセットした半透明グレーの「ドロップシャドウ」。
- ぼかし（blur）は基本ゼロ。常に**くっきりした矩形の影**で、紙が少し浮いているような
  「ペーパークラフト/コラージュ」感を出す。
- 画像、ボタン、スキルタグ、リンクホバーなど、ほぼ全てのインタラクティブ要素・カード要素に
  このシャドウパターンを適用し、デザインの一貫性を保っている。
- ホバー時はシャドウのオフセットが伸びる（6px→8px）＋ `scale(1.1)` で「持ち上がる」感覚を演出。
  クリック時は `scale(0.95)` 程度に縮めてシャドウを短くし「押し込む」感覚を出す。

枠線は実線ではなく**`dashed`（破線）**を多用（区切り線 `hr`、スキルタグの枠、ローディングバーの枠など）。
これも「コピー用紙の切り取り線」「ミシン目」を連想させる手作り感の演出。

---

## 5. レイアウト構造

- **1カラム・中央寄せ・最大幅クランプ**：`width: clamp(0px, 95vw, 680px)` のように
  画面幅に応じて伸縮しつつ上限を設ける手法を多用。モバイルからデスクトップまで同じリズムを保つ。
- セクションごとに**点線の下線（border-bottom: dashed）**で区切り、Zineのページ区切りのように見せる。
- 画像・動画・iframeは `.image` コンテナでフレックス折り返し表示。全要素に統一シャドウが付くため、
  写真がまるで「コラージュに貼り付けたプリント」のように見える。
- フッターは小さい文字（12px）で、著作権表記や先住民への敬意表明など縦に積んだプレーンテキスト。

---

## 6. ナビゲーション（ハンバーガーメニュー）

- 通常時は透明な背景にアイコンボタン（LinkedIn/Bluesky/GitHub + ハンバーガー）が浮かぶだけ。
  スクロールするまでボタンは非表示〜薄い状態（`opacity: 0` → スクロールで `1`）。
- メニューを開くと、画面左から**ダークグレーのパネルがスライドイン**し、
  ベージュのピクセルフォントで大きくリンクが並ぶ「全画面オーバーレイメニュー」形式。
- ボタンは全て前述の二重シャドウ＋ベージュ縁取りのスタイルで統一。

---

## 7. モーション／インタラクションの質感

- トランジションは基本 `0.3s` 前後の短いイージングで、機械的というよりは「カチッ」とした
  物理的なフィードバック感を重視（scaleとshadowの同時変化）。
- ローディング画面はダークグレー全面に、破線枠のプログレスバー（コンピュータの起動画面風）。
  "Booting..." のような文言でレトロPCの起動シーケンスを演出。
- スクロール量に応じて3Dシーンからフラットなセクションへ滑らかに移行する「縦長ヒーロー
  （`margin-top: 200vh`）」構成で、最初の2画面分は3D空間内を覗き込むような体験になる。

---

## 8. デザインの再現に使える要点まとめ（チェックリスト）

新しいサイトでこの「世界観」を再現する場合のポイント：

- [ ] 背景＝暖色クリーム、前景＝中間グレーの2色だけで統一（純白/純黒を避ける）
- [ ] 見出しはピクセル/モノスペースフォント＋背景色反転の「ラベル」表示
- [ ] 本文は読みやすいサンセリフで対比をつける
- [ ] カード/ボタン/画像すべてに「1px縁取り＋6pxオフセットの半透明シャドウ」を統一適用
- [ ] 区切り線・枠線は破線（dashed）を基本にする
- [ ] ホバーで scale up + shadow拡大、クリックで scale down + shadow縮小という
      「物理的に持ち上がる/押し込む」インタラクション
- [ ] 1カラム・`clamp()`で幅を制御した中央寄せレイアウト
- [ ] 起動画面/ローディング演出でレトロコンピュータらしさを補強
- [ ] 大胆な3D/WebGLヒーロー → スクロールでフラットなZine風コンテンツへ、という二層構成
